# 第三方接口调用工具 - 天气查询（get_weather）

## 基本信息

| 属性     | 值                     |
| -------- | ---------------------- |
| 工具名称 | `get_weather`          |
| 工具描述 | 查询指定城市的天气信息 |

## 入参格式

```json
{
	"city": "需要查询天气的城市名称"
}
```

# 参数说明

参数名 | 类型 | 必填 | 说明
city | string | 是 | 需要查询天气的城市名称，如：北京、上海、广州

# 返回值格式

- 成功返回：

```json
{
	"message": "xxx地的天气如何"
}
```

# 失败返回：

```
天气信息获取失败
```

# 第三方接口信息

- 请求地址 http://192.168.1.36:3000/api/test/weather
- 请求方法 POST
- 请求头 {Content-Type: application/json}

# 请求体示例：

```json
{
	"city": "北京"
}
```

# 完整请求示例：

```
POST http://192.168.1.36:3000/api/test/weather
Content-Type: application/json

{
  "city": "北京"
}
```

# 异常处理规则

```
异常类型	处理方式
网络错误	返回 天气信息获取失败
接口报错（非200状态码）	返回 天气信息获取失败
超时	返回 天气信息获取失败
返回数据格式错误	返回 天气信息获取失败
```

# LLM 返回格式要求

强制返回格式： 当用户询问天气时，LLM 必须 返回以下 JSON 格式，不要添加任何额外文字说明：

```
{
  "tool": "get_weather",
  "params": {
    "city": "提取到的城市名称"
  }
}
```

字段 | 类型 | 说明
tool |string | 固定值 get_weather
params | object | 参数对象
params.city | string | 从用户问题中提取的城市名称

# 使用示例

- 用户输入： 北京天气怎么样？

- LLM 输出：

```json
{
	"tool": "get_weather",
	"params": {
		"city": "北京"
	}
}
```

- 实际 API 调用：

```http
POST http://192.168.1.36:3000/api/test/weather
Content-Type: application/json

{
  "city": "北京"
}
```

- API 成功响应：

```json
{
	"message": "北京今天晴天，温度25°C，空气质量良好"
}
```

# 注意事项

- LLM 输出格式严格性：必须确保 LLM 只输出 JSON，不加任何额外文字

- 城市名称提取准确性：LLM 需要正确识别用户提到的城市名称

- 接口可用性：确保 http://192.168.1.36:3000 服务正常运行

- 网络超时：建议设置 10 秒超时，避免长时间等待

- 错误兜底：任何异常都应返回 天气信息获取失败，保证用户体验
